# Assessments

This folder contains various assessments to help you evaluate your understanding of system design concepts and apply them to real-world problems.

## üìã Contents

### 1. Self-Check Quizzes (`self-check-quizzes/`)

**Purpose:** Test your understanding of each module's concepts.

**What's Included:**
- 15 quizzes (one per module)
- 10 questions per quiz
- Mix of multiple choice and short answer questions
- Comprehensive answer key with explanations

**How to Use:**
1. Complete a module from the main course
2. Take the corresponding quiz without looking at answers
3. Check your answers using `answers.md`
4. Review any concepts you struggled with
5. Retake the quiz after a few days to ensure retention

**Time Commitment:** ~20 minutes per quiz

**Files:**
- `module-00-foundations-quiz.md` through `module-14-cdn-quiz.md`
- `answers.md` - Complete answer key for all quizzes

---

### 2. Design Challenges (`design-challenges/`)

**Purpose:** Practice designing complete systems under time constraints.

**What's Included:**
- 5 comprehensive design challenges
- 45-minute time limit per challenge
- Self-grading rubrics
- Detailed sample solutions

**Challenges:**

1. **URL Shortener** (Beginner-Intermediate)
   - Topics: Database design, hashing, caching, scalability
   - Good first challenge for beginners

2. **Social Media News Feed** (Intermediate)
   - Topics: Fan-out patterns, caching, real-time updates
   - Practice handling read-heavy workloads

3. **Distributed Rate Limiter** (Intermediate-Advanced)
   - Topics: Rate limiting algorithms, distributed systems, caching
   - Learn about protecting APIs at scale

4. **Distributed Key-Value Cache** (Advanced)
   - Topics: Consistent hashing, replication, CAP theorem
   - Deep dive into distributed systems

5. **Web Crawler** (Advanced)
   - Topics: Distributed systems, queuing, deduplication
   - Practice designing systems with complex constraints

**How to Use:**
1. Set a timer for 45 minutes
2. Attempt the challenge without looking at the solution
3. Grade yourself using the rubric
4. Compare your design with the sample solution
5. Identify areas for improvement
6. Try redesigning weak areas

**Time Commitment:** 45 minutes per challenge + review time

---

### 3. Final Project (`final-project.md`)

**Purpose:** Demonstrate comprehensive mastery of system design through a capstone project.

**What's Required:**
- Design a complex system incorporating 10+ course concepts
- Written document (15-25 pages)
- Architecture diagrams (5 required)
- Presentation (15-20 slides)

**Project Options:**
1. Real-Time Collaboration Platform (like Google Docs)
2. Food Delivery Platform (like Uber Eats)
3. Video Streaming Platform (like YouTube)
4. E-Commerce Marketplace (like Amazon)
5. Social Media Platform (like Instagram)
6. Custom Proposal (subject to approval)

**Deliverables:**
- Complete design document with capacity estimates, API design, architecture
- Professional diagrams (high-level, data flow, database, deployment, sequence)
- Presentation slides with Q&A preparation

**Time Commitment:** 10-15 hours over 2-3 weeks

**Grading:**
- Written Document: 60 points
- Diagrams: 20 points
- Presentation: 20 points
- Bonus: Up to 10 points
- **Total: 100 points (110 with bonus)**
